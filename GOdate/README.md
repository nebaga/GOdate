  ### GOdate — приложение для свиданий и маршрутов

  GOdate — это веб‑приложение с готовыми маршрутами, избранным, «дейликами», уведомлениями, конструктором маршрутов (Яндекс.Карты) и генерацией маршрутов с помощью ИИ.

  ---

  ### Стек
  - Backend: FastAPI (Python), SQLite (файл `backend/godate.db`)
  - Frontend: статические HTML/CSS/JS (без сборщика)
  - Карты: Яндекс.Карты JS API

  ---

  ### Быстрый старт (Windows, PowerShell)
  1) Установите Python 3.10+ и pip

  2) Установите зависимости backend
  ```
  py -m pip install -r backend/requirements.txt
  ```

  3) Запустите сервер (порт фронта/АПИ: 8876)
  ```
  py -m uvicorn backend.main:app --port 8876
  ```

  4) Откройте приложение
  ```
  http://127.0.0.1:8876
  ```
  Backend сам раздаёт статические файлы из корня проекта. В HTML уже прописан `BASE_URL: 'http://localhost:8876/api'`.

  БД создаётся и инициализируется автоматически (демо‑маршруты/дейлики) при первом старте.

  ### Основные возможности
  - Готовые маршруты с «свайпами»:
    - вправо — добавить в избранное
    - влево — пропустить (до следующего дня в 00:00 МСК не показывается)
  - Избранные маршруты: список + мини‑карта маршрута (точки/путь)
  - Уведомления: заявки в друзья/половинки
  - Дейлики: выполнение 1 раз в сутки, таймер до 00:00 МСК
  - Конструктор маршрутов: ручной ввод координат точек, предпросмотр на Яндекс.Картах
  - Мои маршруты (в профиле): список, удаление, редактирование
  - ИИ‑генерация маршрутов: текст + попытка распознавания координат

  ---

  ### Конфигурация карт (Яндекс.Карты)
  На страницах `index.html`, `routes.html`, `ai.html`, `create-route.html` подключён скрипт:
  ```
  <script src="https://api-maps.yandex.ru/2.1/?apikey=ВАШ_API_КЛЮЧ&lang=ru_RU"></script>
  ```
  Замените `apikey=...` на ваш ключ в этих файлах при необходимости.

  ---

  ### Конфигурация ИИ
  В `backend/main.py` используется внешний сервис `ask.chadgpt.ru`.
  - Ключ `CHAD_API_KEY` захардкожен. При необходимости замените его и/или источник API.

  ### API (кратко)
  Базовый путь: `/api`

  - Auth
    - `POST /auth/register`, `POST /auth/login`
  - Пользователь
    - `GET /users/me`, `POST /users/logout`, `POST /users/avatar`
    - `POST /users/request` (заявки), `DELETE /users/soulmate`, `DELETE /users/friend`
  - Уведомления
    - `GET /messages`, `POST /messages/act`
  - Дейлики
    - `GET /dailies/today`, `POST /dailies/complete`
  - Маршруты (общие/избранные)
    - `GET /routes` (фильтр `?city=`)
    - `POST /routes/favorite`, `GET /routes/favorites`, `DELETE /routes/favorite/{route_id}`
  - Мои маршруты
    - `POST /routes` — создать (через конструктор)
    - `GET /routes/mine` — мои маршруты
    - `GET /routes/{id}` — получить мой маршрут
    - `PUT /routes/{id}` — обновить мой маршрут
    - `DELETE /routes/{id}` — удалить мой маршрут
  - Рейтинг
    - `GET /rating`
  - ИИ генерация
    - `POST /ai/generate`

  Формат ответов описан в `backend/schemas.py`.

  ### Настройки фронтенда
  В каждой странице есть блок конфигурации API:
  ```
  window.API_CONFIG = {
    BASE_URL: 'http://localhost:8876/api',
    ENDPOINTS: { AUTH: '/auth', USERS: '/users', ROUTES: '/routes', ... }
  }
  ```
  Если меняете порт/хост бэкенда — обновите `BASE_URL` во всех страницах.

  ---

  ### Частые вопросы
  - «Статика не открывается через файл»: используйте `http://127.0.0.1:8876` — backend отдаёт статику и избегает ограничений `file://`.
  - «Карта не показывает маршрут»: убедитесь, что у маршрута есть ≥2 точки с валидными координатами.
  - «Дейлик повторно не выполняется»: это норма, он обновится в 00:00 по МСК (есть таймер).
